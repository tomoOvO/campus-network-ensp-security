# 校园网搭建与安全测试

## 📌 项目简介
本项目使用 **华为 eNSP 模拟器**，模拟了某中型校园网的网络架构，涵盖教学楼、宿舍区、行政楼等多个区域。在完成组网后，使用 **Kali Linux** 对内网进行多阶段渗透测试，发现并验证了关键漏洞，最终输出了一套可落地的安全加固方案。

## 🛠️ 技术栈
- **网络模拟**：eNSP
- **渗透测试**：Kali Linux, Nmap, Metasploit (MS17-010)
- **网络协议**：VLAN, OSPF, ACL, DHCP, STP
- **分析工具**：Wireshark

## 🏗️ 网络拓扑设计
- **区域划分**：教学区 VLAN 10、宿舍区 VLAN 20、服务器区 VLAN 30、管理区 VLAN 40
- **核心层**：三层交换机互联，运行 OSPF 动态路由
- **出口**：路由器连接外网，配置 NAT 和 ACL 访问控制
- **用户服务**：部署 DHCP 自动分配 IP，所有接入层交换机启用端口安全

![网络拓扑图](topology/topology_overview.png)

## 🚀 主要功能
- **全互联互通**：不同 VLAN 间通过三层交换实现路由可达
- **安全基础**：ACL 禁止宿舍区直接访问服务器区，只允许特定管理 IP
- **冗余设计**：核心层使用双链路聚合，STP 防止环路
- **地址管理**：DHCP 服务器按 VLAN 动态分配地址

## 🧪 渗透测试过程
1. **信息收集**：在 Kali 上使用 Nmap 扫描内网段，发现存活主机及开放端口
2. **漏洞利用**：针对一台未打补丁的 Windows 7 主机，使用 Metasploit 的 `ms17-010_eternalblue` 模块成功获取 SYSTEM 权限
3. **后渗透**：加载 Mimikatz 尝试抓取明文密码，验证横向移动可行性
4. **流量分析**：在核心交换机镜像端口抓包，对比正常与攻击流量特征

## 🛡️ 发现的安全问题与加固
| 发现问题 | 风险级别 | 加固措施 |
|---------|--------|---------|
| 未隔离高风险端口（445） | 高 | ACL 阻断跨区域 445 端口流量 |
| 存在系统未更新补丁 | 高 | 部署 WSUS 补丁管理，强制更新 |
| DHCP 无防护 | 中 | 启用 DHCP Snooping 防欺骗 |
| 管理用 Telnet 明文 | 中 | 切换为 SSH，配置 ACL 限制管理源 IP |
| 内网无流量监控 | 低 | 部署 SNMP+Zabbix 基础监控 |

完整加固报告见：[安全加固方案](hardening/security_hardening_plan.md)

## 📊 项目成果
- 掌握大中型园区网络规划与排错能力
- 深入理解 OWASP Top 10 之外的网络层攻击与防御
- 独立完成从组网、渗透到加固的闭环实践
