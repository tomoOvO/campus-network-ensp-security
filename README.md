# 校园网综合建设与安全评估

## 📌 项目简介
本项目使用 **华为 eNSP** 模拟器，严格按照《校园网建设项目需求书》完成了 **10 项核心网络功能**的搭建。在此基础上，引入 **Kali Linux** 进行多阶段渗透测试，使用 **Wireshark** 对比分析攻击前后流量，通过 **SNMPb** 实现设备实时监控，并利用 **PuTTY** 完成所有设备的远程管理。此外，针对 7 种常见网络攻击行为，编写了完整的 **应急预案报告**，形成“规划设计 → 服务部署 → 攻击测试 → 应急响应 → 安全加固”的全流程闭环实践。

## 🛠️ 综合技术栈
- **网络模拟与搭建**：eNSP
- **渗透测试**：Kali Linux, Nmap, Metasploit (MS17-010), Hydra
- **攻击复现**：ARP欺骗、DHCP饥饿、MAC洪泛、SSH爆破、SYN洪水、端口扫描、目录扫描
- **流量分析**：Wireshark
- **设备监控**：SNMPb (SNMP 网络管理软件)
- **远程管理**：PuTTY (Telnet / SSH)
- **网络协议**：VLAN, Trunk, OSPF, DHCP, NAT, ACL, SNMP

## 🚀 十项核心功能实现详情

### 1. VLAN 与 Trunk 建设
- 在接入交换机上按端口划分 VLAN，终端接口设为 Access 模式
- 交换机间互联口配置为 Trunk，仅允许所需 VLAN 通过，有效缩小广播域
- 核心交换机通过 Trunk 汇聚全部 VLAN 流量

### 2. 网关与 VLAN 间路由
- 核心三层交换机启用各 VLAN 对应的 `Vlanif` 接口，配置网关地址
- 依靠三层交换机的直连路由，实现所有 VLAN 间的无阻互通
- 测试：任意跨 VLAN 终端互 Ping 可达

### 3. DHCP 自动分配
- 在核心交换机上配置 DHCP 服务，按 VLAN 创建独立地址池
- 每个地址池下发网关、DNS（8.8.8.8）及租期
- 终端启用 DHCP 后，可自动获取完整网络配置
- **Wireshark 验证**：在接入端口抓取 DHCP Discover-Offer-Request-ACK 四次握手过程

### 4. OSPF 动态路由
- 新老校区核心交换机运行 OSPF 协议，同属 Area 0
- 宣告各 VLAN 网段及互联接口网段，自动学习全网路由
- 人为断开某条核心链路，OSPF 快速收敛，通信恢复，验证冗余能力

### 5. 出口多链路选路
- 出口路由器配置两条默认路由，指向不同 ISP，其中备份链路使用较高的 `preference` 值
- 结合 NQA（网络质量分析）检测主链路连通性
- 主链路故障时自动切换至备份链路，恢复后切回，切换过程使用 **Wireshark** 抓包观察 NAT 地址变化

### 6. NAT 地址转换（NAPT）
- 在出口路由器连接 ISP 的接口上配置 `nat outbound`，实现内网地址到公网接口地址的转换
- ACL 精准匹配允许上网的内网网段（禁止二号学院楼网段）
- **验证**：Kali Linux 模拟外网主机，抓包查看源 IP 已变为公网地址

### 7. Web 服务器映射（NAT Server）
- 将内网 Web 服务器（192.168.30.100）的 80 端口映射到出口路由器公网地址的 8080 端口
- 配置 `nat server` 实现入向映射
- **测试**：从 Kali 外网虚拟机直接访问 `http://公网IP:8080`，成功打开内网网页

### 8. 设备远程管理（Telnet + PuTTY）
- 所有交换机、路由器启用 Telnet 服务，配置 VTY 线路使用统一认证账号 `admin / campus123`
- 使用 **PuTTY** 从管理区 VLAN 40 的主机远程登录任意设备，执行查看、配置操作
- **安全加固后**：将 Telnet 切换为 SSH，并限制管理源 IP（详见加固方案）

### 9. 访问控制策略（ACL）
- **仅行政楼可访问财务服务器**：在财务服务器所在 VLAN 30 的网关接口应用入向 ACL，仅放行源 IP 192.168.40.0/24
- **禁止二号学院楼访问外网**：在出口路由器 NAT 的 ACL 中拒绝 192.168.50.0/24
- **渗透测试验证**：从 Kali 模拟二号学院楼主机尝试访问外网和财务服务器，均被正确阻断

### 10. 设备监控（SNMP + SNMPb）
- 在所有交换机、路由器上启用 SNMP v2c，配置只读团体字 `public`
- 在管理主机上安装 **SNMPb** 网络管理软件，添加核心设备
- 实时监控设备 CPU、内存、端口流量、UP/DOWN 状态

## 🚨 应急响应与防御演练
针对校园网常见的 7 类攻击行为，从**攻击复现、特征分析、应急处置、长期加固**四个维度编写了标准化应急预案报告，均在隔离实验环境中完成验证。

| 序号 | 攻击类型 | 攻击手段 | 应急措施摘要 | 完整报告 |
|------|---------|---------|-------------|---------|
| 1 | **ARP 欺骗** | 伪造 ARP 响应，实施中间人攻击 | 绑定核心设备静态 ARP、开启 DAI 检测、划分 VLAN 缩小影响面 | [ARP欺骗应急预案](incident_response/arp_spoofing.md) |
| 2 | **DHCP 饥饿攻击** | 伪造大量 DHCP 请求耗尽地址池 | 启用 DHCP Snooping + 端口安全，限制每个端口最大 MAC 数量 | [DHCP饥饿应急预案](incident_response/dhcp_starvation.md) |
| 3 | **MAC 洪泛攻击** | 发送大量虚假 MAC 地址占满交换表 | 配置端口安全、Sticky MAC，超出阈值后 shutdown 或 restrict | [MAC洪泛应急预案](incident_response/mac_flooding.md) |
| 4 | **SSH 暴力破解** | Hydra 对交换机 SSH 进行字典攻击 | 配置 VTY 线路 ACL 限制管理源 IP，设置登录失败锁定与延迟，改用密钥认证 | [SSH暴力破解应急预案](incident_response/ssh_brute_force.md) |
| 5 | **TCP SYN 洪水攻击** | 利用半开连接消耗服务器资源 | 出口路由器配置限制 SYN 速率，服务器开启 SYN Cookie，部署防火墙阈值告警 | [SYN洪水应急预案](incident_response/syn_flood.md) |
| 6 | **端口扫描** | Nmap 进行全端口与服务探测 | 关闭非必要服务，部署入侵检测规则，ACL 限制扫描源 IP | [端口扫描应急预案](incident_response/port_scan.md) |
| 7 | **目录扫描** | Dirb/gobuster 暴破 Web 目录 | Web 服务器配置限制访问频率、隐藏版本信息，部署 WAF 规则 | [目录扫描应急预案](incident_response/directory_scan.md) |

每份报告均包含：实验拓扑、攻击复现命令、Wireshark 抓包特征、应急处置流程、长期加固方案以及验证测试记录。

## 🛡️ 安全加固方案（结合发现的问题）

| 漏洞/风险 | 发现方式 | 加固措施 |
|----------|--------|---------|
| 未隔离高危端口（445） | Kali 扫描 + 利用成功 | 在 VLAN 30 入向 ACL 拒绝 445 端口，仅在必需时放行 |
| 存在未修补的永恒之蓝漏洞 | Metasploit 利用 | 所有 Windows 主机更新补丁，部署 WSUS 集中管理 |
| Telnet 明文传输密码 | Wireshark 抓包可见 | 所有设备改用 SSH 远程管理，ACL 限制管理源 IP |
| DHCP 无防护 | DHCP 饥饿攻击复现成功 | 启用 DHCP Snooping 及信任端口，配置端口安全 |
| SNMP 使用 v2c 弱团体字 | SNMPb 可随意获取信息 | 升级为 SNMPv3，配置认证和加密 |
| 财务服务器无额外防护 | 仅 ACL 隔离 | 增加登录审计、限制登录源、开启防火墙 |
| ARP 无防护 | ARP 欺骗攻击成功 | 核心设备绑定静态 ARP，接入层开启 DAI |
| 设备登录易受暴力破解 | SSH 爆破测试 | 配置登录失败限制、改用密钥认证、部署堡垒机 |
